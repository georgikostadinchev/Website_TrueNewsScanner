1. Keep the existing API response shape unchanged
2. Keep TypeScript
3. Do not rewrite the frontend
4. Do not remove AI support
5. Make the final score deterministic in backend

Implement these improvements:

1. Central scoring engine
    Create a reusable scoring module, for example:

* artifacts/api-server/src/lib/scoring/engine.ts
* artifacts/api-server/src/lib/scoring/types.ts

The scoring engine should combine:

* heuristic signals
* AI signals
* community report signals
* exact-match memory signals
* trusted allowlist signals

It must output:

* riskScore
* confidence
* verdict
* verdictLabel

Use these thresholds:

* 0–19 safe
* 20–39 insufficient
* 40–59 suspicious
* 60–79 misleading
* 80–100 scam

2. AI should be advisory, not final authority
    Refactor so AI no longer fully decides the final result.
    Minimal acceptable version:

* keep the current AI call
* treat AI riskScore as one weighted signal
* merge it with deterministic signals
* let the new scoring engine compute the final score

3. Connect community reports to /check
    Use reportsTable for matching the checked input:

* exact match first
* only reviewed reports should strongly affect risk
* pending reports should have little or no influence
* dismissed reports should reduce confidence or neutralize report impact

4. Add exact-match memory
    For url and phone:

* normalize input consistently
* reviewed report matches should strongly influence risk
* previous check matches may add a weak consistency signal only
* do not treat previous checks as ground truth

Normalization:

* phone: remove spaces, dashes, parentheses
* url: trim, lowercase host, safely remove trailing slash for comparison

* gov.bg

* nap.bg
* nsi.bg
* mvr.bg
* lex.bg
* bta.bg
* bnr.bg
* bntnews.bg

Allowlist behavior:

* reduce risk
* slightly improve confidence
* never force guaranteed safety by itself

6. Dynamic confidence
    Replace static confidence with calculated confidence based on:

* number of independent signals
* agreement between AI and heuristics
* presence of reviewed reports
* exact-match strength
* allowlist presence
* conflicting evidence
